# Use the ANTLR listener, not the visitor, to lower GQL parse trees
We walk the generated GQL parse tree with the ANTLR **listener** (`ParseTreeWalker` + `BaseGQLListener`), overriding only the handful of `Enter*` rules we care about, rather than the visitor.
## Considered options
The visitor seems more powerful — it returns values up the tree and lets you control traversal. But ANTLR-Go has no virtual dispatch: the generated `BaseGQLVisitor` defaults call `BaseParseTreeVisitor.VisitChildren`, a no-op, so the visitor does **not** auto-recurse. Our points of interest (node/edge/property contexts) sit many rule-levels below `gqlProgram` in a ~60k-line generated parser, so a visitor forces us to hand-roll the tree walk — which is exactly the mess this work replaces. The walker descends the whole tree automatically and fires only our overrides, regardless of depth.

The extra power of the visitor (return-value composition, conditional descent, revisiting) is interpreter/evaluator power. gqlc is an sqlc analogue: a code generator that _extracts_ a model from the tree and never evaluates it. The visitor-vs-listener choice only governs the parse-tree → model boundary; every downstream pass runs over our own structs where ANTLR patterns don't apply. Listener+stack can build anything a visitor can, so nothing is lost.
## Consequences
The traversal is driven by the listener's `ParseTreeWalker`, which cannot be stopped mid-walk — but no per-rule guard is needed to cope with that. `fail()` records the first sentinel and is idempotent, and `Parse` discards the collected pieces (never calling `resolve`) the moment an error is set. So an `Enter*` that runs after the first error just appends data that is thrown away — harmless — and the handlers stay plain, with nothing to remember to add. Fail-fast semantics are preserved: the first error in walk order is the one returned, and for schema-sized inputs the traversal wasted after an error is negligible. We deliberately did not reach for `panic`/`recover` to abort the walk early; it would reverse the sentinel/error-listener convention (AGENTS.md) to save a traversal cost that is already negligible. A previous iteration used the visitor and had to reimplement the walker by hand; this reverses that.
